┌─────────────────────────────────────────────────────────────────────────────┐
│  ЩО ЦЕ ВЗАГАЛІ?                                                              │
│  • До чату: за бажанням — блок «Welcome agents» у сайдбарі: ім’я, мова,    │
│    кнопка Start генерує два короткі повідомлення з сесійних JSON (див. нижче).│
│  • У чаті: користувач пише повідомлення. У сайдбарі обрано «режим» (інтент). │
│  • За потреби завантажені два JSON-файли (session і steps) — вони йдуть у     │
│    нейромережу як дані для аналізу. Всі відповіді йдуть через OpenRouter.    │
└─────────────────────────────────────────────────────────────────────────────┘


┌─────────────────────────────────────────────────────────────────────────────┐
│  ЩО ТАКЕ «АГЕНТ» ТУТ?                                                        │
│  Це не окремі програми. Це різні РОЛІ виклику моделі через OpenRouter:       │
│  • «Класифікатор» (intent_classifier) — тип повідомлення в чаті.            │
│  • «Сесійний помічник» (session_question) — відповідь по JSON у чаті.         │
│  • «Персоналізація завершення» (completion_personalization) — до чату:       │
│    тепле привітання з контексту completion_rationale.                        │
│  • «Залучення до діалогу» (session_engagement) — до чату: один рядок із       │
│    цікавими фактами з session (+ steps), без привітання та без імені;         │
│    щоб схилити до подальшої розмови.                                          │
│  Різниця — у тексті промпта (завдання, правила) та у обраній моделі.         │
└─────────────────────────────────────────────────────────────────────────────┘


┌─────────────────────────────────────────────────────────────────────────────┐
│  ВХІДНІ ДАНІ (нагадування)                                                    │
│  • API-ключ   — у Streamlit Secrets (OPENROUTER_API_KEY).                   │
│  • Session   — довільний JSON з 1-го файлу (йде в контекст як «session»).  │
│  • Steps     — довільний JSON з 2-го файлу (йде як «steps»).                │
│  • Останні 6 повідомлень діалогу — для класифікатора та сесійного агента.   │
│  • Pending skill question — опційно: після відповіді «поставити» ще питання.│
│  • Welcome-блок: ім’я (лише для агента персоналізації completion), мова       │
│    виводу (english / german / ukrainian / russian).                          │
└─────────────────────────────────────────────────────────────────────────────┘


================================================================================
  ПОТІК «WELCOME» (КНОПКА START, ДО ПОВІДОМЛЕННЯ В ЧАТ)
================================================================================

  Умови:
  • Завантажений валідний Session (JSON).
  • У JSON є масив(и) `goals`: береться перший запис у порядку списку, де
    `goal_completion_status` дорівнює «complete» (без урахування регістру),
    і з нього читається `completion_rationale` (непорожній рядок).
  • Якщо такого запису немає — Start показує помилку, виклики LLM не йдуть.

  Порядок викликів:

================================================================================
  ТРИ РЕЖИМИ В САЙДБАРІ (ПРОСТИМИ СЛОВАМИ)
================================================================================

1)  session_question
    ────────────────
    «Я вже знаю, що питання стосується сесії / звіту.»
    Нейромережа викликається ОДИН раз — одразу «сесійний агент».
    Підходить, коли не хочеш зайвого кроку з класифікатором.

2)  general_chat
    ────────────
    «Я хочу, спочатку визначити тип питання автоматично.»
    Спочатку — класифікатор. Якщо він вважає, що треба аналізувати дані
    (session_question), додається ДРУГИЙ виклик — сесійний агент.
    Якщо класифікатор каже general_chat — у чаті тільки рядок з міткою
    (окремого «болтуна»-агента в демо немає).

3)  skill_creation
    ───────────────
    «Режим для створення скілла» — у демо НЕ викликається модель, лише
    текст, що логіку треба налаштувати в бекенді. Це заглушка.


================================================================================
  ДЕ «ЖИВУТЬ» ПРОМПТИ (У КОДІ)
================================================================================

• Класифікатор:              build_intent_classifier_messages
• Сесійний агент (чат):      build_session_messages
• Персоналізація completion: build_completion_personalization_messages
• Залучення (факти з сесії): build_session_engagement_messages
• Усі виклики HTTP:          call_openrouter (OpenAI-сумісний /chat/completions
  через https://openrouter.ai/...; тіло — JSON, токен в заголовку).

Моделі в демо:
• Чат (вибір у сайдбарі «Model») — зазвичай з пресетів або custom.
• Welcome агент 1 — фіксовано openai/gpt-4o-mini (константа в коді).
• Welcome агент 2 — фіксовано google/gemini-2.5-pro (константа в коді).


================================================================================
  ДОДАТКО: Pending skill question
================================================================================

Якщо це поле в сайдбарі не порожнє, у промпт сесійного агента додається
правило: після відповіді на питання користувача «м’яко перейти» і задати
ще одне, зазначене тобою питання (сценарій на кшталт «продовження skill»).
